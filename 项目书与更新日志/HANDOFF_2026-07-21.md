## 二、本次翻新涉及的架构变更（关键！）

### 2.1 存储层
- `src/services/sqlite/index.ts`：新增 `messages` 表 + `runSql`/`querySql`/`runSqlNoSave`/`runInTransaction`
- `src/services/chat-message-db.ts`：消息分页查询 + 搜索 + CRUD + blob 迁移
- **双写策略**：`addMessage` 同时写 store（blob 持久化）和 messages 表（查询用）
- `usePersistence`：启动时自动从 blob 迁移到 messages 表

### 2.2 聊天渲染
- `ChatView.tsx`：**完全重写**。窗口化渲染，Store 为主数据源（纯内存），DB 为辅。`subscribe` 无条件触发 → `syncFromStore` 从 store 读取最新内容。
- **关键教训**：流式更新只改 `content` 不改数组长度 → subscribe 必须监听全部变化而非仅长度。参考 RikkaHub：流式阶段不写 DB，纯内存 StateFlow 驱动 UI。

### 2.3 代码高亮
- `src/services/highlight/index.ts`：hljs 封装 + 复制按钮
- `MarkdownRenderer.tsx`：**useMemo + dangerouslySetInnerHTML** 在渲染阶段高亮（非 useEffect），避免 React Virtual DOM 覆盖。代码块浅色主题背景（`var(--app-secondary)`）。

### 2.4 消息交互
- `MessageContextMenu.tsx`：桌面右键 / 移动端 400ms 长按
- `MessageEditDrawer.tsx`：底部滑出窗口，查看/编辑 Markdown 原文
- `ExportDrawer.tsx`：底部抽屉，勾选消息→Markdown 下载
- `ChatView.tsx`：编辑三写（windowMessages + chat-store + chat-message-db）

### 2.5 后台任务
- `src/services/background-task/index.ts`：`BackgroundTaskManager` 单例
- `use-send-message.ts`：注册为后台任务
- `App.tsx`：监听非活跃对话完成→推送通知

### 2.6 新增字段
